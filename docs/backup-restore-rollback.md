# 备份恢复回滚策略

## 目标

- 备份可完整导出：`sessions`、`settings`、`icon_cache`。
- 恢复可安全覆盖：恢复失败时不破坏现有库。
- 恢复后立即生效：托盘行为、自启动行为与追踪视图同步刷新。

## 备份格式

- 文件类型：`*.ttbackup.json`
- 结构：
  - `version`
  - `meta`（导出时间、schema 版本、应用版本）
  - `sessions`
  - `settings`
  - `icon_cache`

## 恢复流程

1. 读取并校验备份文件版本。
2. 开启数据库事务。
3. 依次清空 `sessions`、`settings`、`icon_cache`。
4. 写入备份中的三类数据。
5. 提交事务；若任何步骤失败则自动回滚。

## 回滚保证

- 所有写入都在单事务内执行。
- 任一 SQL 失败或提交失败，事务会回滚到恢复前状态。
- 前端提示“已自动回滚，不会破坏当前数据”。

## 高风险确认文案

统一使用：

- 标题：`【高风险操作】<动作>`
- 主体：`此操作不可撤销，是否继续？`

用于：

- 恢复备份
- 历史数据清理
- 删除应用全部历史记录

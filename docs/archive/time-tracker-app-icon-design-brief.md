# Time Tracker 应用图标设计 Brief

## 1. 文档定位

本文件用于定义 `Time Tracker` 的产品级应用图标设计目标、边界、评审标准与推荐方向。

它不是一次性执行单，也不是某一轮概念图探索的记录，而是后续图标设计、图标迭代、导出应用资产时的长期 brief。

本文件优先级高于当前 `src-tauri/icons/*` 中已有的历史图标资产。

换句话说：

- 现有 `src-tauri/icons/*` 只是当前打包用资产
- 它们不是产品品牌来源
- 后续若与本 brief 冲突，应以本 brief 为准

---

## 2. 与其他长期文档的关系

本文件主要回答：

- `Time Tracker` 的产品图标到底应表达什么
- 什么样的气质符合当前产品
- 哪些方向必须主动排除
- 图标系统最终应该交付哪些资产

它与其他长期文档的关系如下：

- 与 [`quiet-pro-component-guidelines.md`](./quiet-pro-component-guidelines.md) 互补，`Quiet Pro` 规定整个产品的长期 UI 气质，本文件规定产品标识的专项设计边界。
- 与 [`architecture-target.md`](./architecture-target.md) 互补，架构文档约束代码边界，本文件约束品牌图标资产与 UI 标识的长期方向。

---

## 3. 设计任务定义

当前需要的不是“从已有 Tauri 图标里挑一个放进 UI”，而是：

**为 Time Tracker 设计真正的产品应用图标系统。**

本次质量目标应明确为：

**高端、成熟、产品级的图标质感。**

这里的“百万设计感”指的是长期可用的产品判断力，不是炫技、潮流化或营销化视觉刺激。

这套图标系统后续应同时服务于：

- 应用内左上角的 `app mark`
- 桌面窗口图标
- 任务栏 / 托盘图标
- 安装包图标
- GitHub Release / 官网 / 文档中的产品识别

因此，正确的关系应是：

- 产品图标先确定
- Tauri 图标资产随后根据产品图标导出

而不是让产品图标去适配已有 Tauri 图标。

---

## 4. 当前产品语义

`Time Tracker` 的图标应传达以下核心感受：

- 时间正在被持续记录
- 活动在后台被平稳追踪
- 产品是安静、专业、长期可靠的
- 它是一个桌面工作伴随型工具，而不是强刺激的效率游戏

图标语义应更接近：

- 时间流
- 记录轨迹
- 持续追踪
- 节律与稳定

而不是更接近：

- 冲刺
- 电力 / 加速
- 安全防护
- 系统监控告警
- 娱乐化 productivity

---

## 5. Quiet Pro 约束下的图标气质

图标必须符合当前仓库的 `Quiet Pro` 长期基线：

- 克制
- 专业
- 清晰
- 不花哨
- 小尺寸可识别

因此必须避免：

- 过强渐变
- 玻璃感
- 霓虹高亮
- 复杂描边叠层
- 依赖细碎细节才能辨认的造型
- 过度拟物

图标风格应偏：

- 几何
- 简洁
- 有节奏感
- 低噪音

在 Quiet Pro 语境下，“高级感”应来自：

- 比例关系
- 形体节奏
- 几何控制
- 细节精度
- 留白自信

而不是来自：

- 强渐变
- 强光效
- 复杂装饰
- 为了“惊艳”而增加的视觉噪音

---

## 6. 评审标准

为保证“百万设计感”可执行，设计评审至少应满足以下标准。

### 6.1 轮廓识别

- 不依赖细碎细节
- 远看和小看都能快速识别
- 第一眼先记住外轮廓，而不是局部装饰

### 6.2 小尺寸成立

- 16px~32px 不糊、不挤、不丢主体关系
- 不因为细节消失就退化成普通时钟或普通圆环

### 6.3 去色后仍成立

- 单色版仍保留主体语义与品牌辨识
- 方案不应依赖颜色才能成立

### 6.4 留白自信

- 不过度填满图形边界
- 中心与边缘呼吸均衡
- 不通过“塞满”来制造存在感

### 6.5 结构克制

- 构件数量受控
- 不做无必要叠层
- 没有任何一处只是为了装饰存在

### 6.6 长期耐看

- 不依赖短期潮流风格
- 1~2 年后仍显专业
- 可以同时服务桌面产品环境与品牌识别场景

---

## 7. 明确排除的方向

以下方向不建议作为 `Time Tracker` 的正式产品图标主方向：

- 闪电  
  原因：太像电源、性能加速、提速器、充电、系统工具

- 纯时钟 / 闹钟 / 秒表  
  原因：过于通用，辨识度不足，也容易误解为番茄钟或定时器

- 柱状图 / 折线图 / 饼图  
  原因：更像数据面板功能，不像产品身份

- 齿轮 / 设置滑杆  
  原因：更像“设置页”或“系统工具”，不适合作为产品主标识

- 魔法棒 / 星星 / 闪粉风格  
  原因：偏装饰或自动化感，不够稳定，不够长期

- 窗口 + 时钟的复杂拼贴  
  原因：结构太复杂，小尺寸下容易糊

---

## 8. 推荐设计方向

建议从以下 3 个方向中探索，而不是从历史 Tauri 图标倒推。

### 方向 A：时间环 / 记录环

关键词：

- 环
- 轨迹
- 时间流
- 留白

语义：

- 时间在持续流动
- 记录行为围绕时间展开

适合：

- 想要非常稳、非常耐用的主图标

风险：

- 如果只做成普通圆环，容易过于抽象

### 方向 B：时间刻度 / 节点轨迹

关键词：

- 刻度
- 节点
- 路径
- 节律

语义：

- 活动被连续采样与归档
- 追踪是“点到线”的形成过程

适合：

- 想强调“追踪”而不是“报表”

风险：

- 结构如果太复杂，小尺寸会丢失辨识
- 稍微做过头就会滑向表盘或通用时间 App

### 方向 C：时间片段 / 分段记录

关键词：

- 片段
- 分段
- 记录块
- 时间切片

语义：

- 时间被分段整理
- 应用活动被结构化记录

适合：

- 想更明确表达“时间追踪软件”这个属性

风险：

- 如果太像图表，就会偏功能图标

---

## 9. 推荐主方向

当前最推荐优先探索：

**方向 A：时间环 / 记录环**

原因：

- 最符合 Quiet Pro 的长期耐看属性
- 最容易在小尺寸下保持清晰
- 不会误导成“设置”“性能”“报表”“番茄钟”
- 适合同时导出 UI mark、窗口图标、任务栏图标与安装图标

建议语义强化方式：

- 不要只是一个完整闭环，也不要做成通用时钟
- 优先使用“开口环 + 记录节点 / 轻刻度”的组合，表达持续记录中的时间
- 切口与节点应服务语义，不做装饰性堆叠

这意味着：

- 环是主角
- 节点和刻度是次识别点
- 识别重点应来自整体构图，而不是某个局部亮点

## 10. 资产体系要求

最终不应该只做“一个图片”，而应该产出同源图标系统。

### 10.1 App Icon 主版

用于：

- 安装包
- 桌面快捷方式
- 任务栏 / 窗口图标
- `src-tauri/icons/*` 导出来源

要求：

- 32px 仍可辨认
- 256px 以上也不过于空洞

### 10.2 UI App Mark 简版

用于：

- 应用左上角标识位
- About 区块
- 未来自定义标题栏中的产品标识

要求：

- 比主版更简洁
- 与主版同源
- 放进 40px Quiet Pro 小面板中不显拥挤

### 10.3 单色版本

用于：

- 托盘
- 小尺寸高对比环境
- 文档和特殊背景场景

要求：

- 去色后仍成立
- 不依赖背景效果

---

## 11. 色彩要求

图标色彩应满足：

- 主色不超过 2 个
- 优先采用 Quiet Pro 兼容色域
- 在浅背景与深背景都能稳定识别

推荐：

- 冷静蓝系为主
- 如需第二色，应是低噪音的辅助色，而非高饱和撞色

不建议：

- 强荧光配色
- 彩虹式多色
- 高反差霓虹

长期上，图标方案应优先先通过单色评审，再决定彩色方案，而不是反过来。

---

## 12. 小尺寸可用性要求

图标设计必须在以下尺寸下可用：

- 16px
- 20px
- 24px
- 32px
- 40px
- 128px
- 256px

尤其要验证：

- 左上角 40px 容器中是否清晰
- 任务栏 16~24px 中是否还能识别主体轮廓

如果某个方案必须依赖大尺寸细节才能成立，那它不适合作为正式产品图标。

---

## 13. 当前左上角位置的使用规则

应用内左上角当前位置应定义为：

**产品标识位**

因此：

- 这里放应用自己的 `app mark`
- 不放页面功能图标
- 不放系统工具类图标
- 不放与某个 feature 绑定的语义图标

侧边栏图标继续承担：

- 页面导航识别

所以左上角标识与侧边栏图标，本来就不应该相同。

---

## 14. 当前旧 Tauri 图标的处理原则

`src-tauri/icons/*` 中的现有图标资产：

- 可以作为历史参考
- 但不作为默认基础
- 如果后续最终品牌方向与它完全不同，可以整体替换

因此，后续执行时不要默认要求“保留旧 Tauri 图标结构”。

---

## 15. 交付顺序建议

正式进入执行前，建议按以下顺序推进：

1. 基于本 brief 产出 2 到 3 个高质量产品图标概念方向
2. 按“小尺寸识别 + 去色可用 + Quiet Pro 一致性”筛选并选定一个主方向
3. 定稿 `app icon + app mark + monochrome` 三种同源版本
4. 再替换 `src-tauri/icons/*`
5. 最后再接入应用内左上角标识位

这个顺序非常重要：

- 先定产品图标
- 再定导出资产
- 最后才做接入

不要反过来先改 Tauri 图标或先改 UI 再回头定义品牌。

---

## 16. 给后续执行者的提示

后续执行时，请记住：

- 不要把当前 `src-tauri/icons/*` 视为品牌真源
- 先做 `Time Tracker` 的产品图标
- 再让 Tauri 图标资产与 UI 标识跟随产品图标统一
- 若某个方向只能通过颜色、纹理或装饰成立，应视为方向本身不够好
- 若某个方向在单色、小尺寸、托盘场景下失效，应主动淘汰

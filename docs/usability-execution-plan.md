# 产品体验人性化改造实施方案（执行版）

## 1. 目标与范围

本方案用于把当前“功能可用”的时间追踪器，升级为“长期常驻可用、低心智负担、用户可控”的桌面应用体验。

目标：

- 常驻体验一致：最小化、关闭、自启动、托盘行为可预测。
- 用户控制完善：可暂停追踪、可排除应用、可快速恢复。
- 隐私与安全可控：敏感信息可关闭采集，数据可备份恢复。
- 交互成本更低：关键入口集中、默认策略合理、误操作可回退。

不在本阶段范围：

- 商业化能力（账户体系、云同步、订阅）
- 多平台扩展（macOS/Linux）
- 外部团队协作功能

## 2. 核心原则

- 默认安全：默认配置优先保护隐私与数据安全。
- 用户可控：关键行为必须有显式开关，不做隐式强制。
- 可撤销：高风险操作（删除、恢复覆盖）必须二次确认。
- 一致性：设置、托盘、窗口按钮三处行为必须一致。
- 最小打扰：不在非必要场景弹窗打断用户。

## 3. 当前基线

已具备：

- 追踪主链路稳定（AFK、锁屏/睡眠边界、watchdog）
- 应用映射与分类能力（重命名、分类、颜色、自定义）
- 历史数据清理能力（按时间范围）

体验缺口：

- [ ] 开机自启动
- [ ] 系统托盘与托盘菜单
- [ ] 关闭/最小化行为策略配置
- [ ] 全局暂停/恢复追踪
- [ ] 应用级“不统计”开关
- [ ] 隐私模式（不记录窗口标题）
- [ ] 数据备份与恢复
- [ ] 首次引导（默认行为选择）

## 4. 阶段计划（必须勾选）

## 阶段 A：常驻体验基础

- [ ] 增加系统托盘图标与托盘菜单（打开主界面、暂停/恢复、退出）
- [ ] 增加 `关闭按钮行为` 设置：`退出应用` / `最小化到托盘`
- [ ] 增加 `最小化行为` 设置：`任务栏` / `托盘`
- [ ] 增加 `显示托盘图标` 开关
- [ ] 增加 `开机自启动` 与 `启动时最小化` 设置

验收门槛：

- [ ] 关闭与最小化行为严格符合设置值
- [ ] 托盘菜单可稳定唤起主窗口与退出
- [ ] 自启动生效且不抢焦点（开启静默启动时）

## 阶段 B：追踪可控与隐私

- [ ] 增加全局 `暂停追踪` 开关（设置页 + 托盘 + 顶部状态）
- [ ] 暂停期间禁止写入新 session，恢复后正常继续
- [ ] 在应用映射页增加 `不统计此应用` 开关
- [ ] 增加 `记录窗口标题` 开关（关闭后不写 `window_title`）
- [ ] 增加“清空历史窗口标题”能力

验收门槛：

- [ ] 暂停期间数据库无新追踪记录
- [ ] 被排除应用不再进入统计口径
- [ ] 关闭标题采集后新数据无标题内容

## 阶段 C：数据安全与恢复

- [ ] 增加 `导出备份`（数据库 + 元数据）
- [ ] 增加 `从备份恢复`（覆盖前二次确认）
- [ ] 恢复失败回滚策略文档化
- [ ] 高风险操作统一确认文案与不可撤销提示

验收门槛：

- [ ] 备份文件可在新环境恢复成功
- [ ] 恢复后应用可正常启动并显示历史数据
- [ ] 恢复失败不破坏原有数据

## 阶段 D：引导与体验收口

- [ ] 新增首次引导：托盘策略 + 关闭行为 + 是否自启动
- [ ] 增加 tracker stale 轻提示（不打断）
- [ ] 统一设置页信息架构（追踪、常驻、隐私、数据安全）
- [ ] 增加关键操作完成反馈（toast/状态提示）

验收门槛：

- [ ] 首次使用用户可在 1 分钟内完成关键设置
- [ ] 无“点关闭后程序消失”类误解反馈
- [ ] 异常状态可见但不干扰日常使用

## 5. 设置项契约（settings 表）

建议新增并统一落库到 `settings(key, value)`：

- `launch_at_login`: `0|1`
- `start_minimized`: `0|1`
- `show_tray_icon`: `0|1`
- `close_behavior`: `exit|tray`
- `minimize_behavior`: `taskbar|tray`
- `tracking_paused`: `0|1`
- `capture_window_title`: `0|1`
- `onboarding_completed`: `0|1`

## 6. 交互决策矩阵（强制统一）

当 `close_behavior=tray`：

- 点窗口关闭按钮：隐藏到托盘，不退出进程
- 托盘菜单“退出应用”：真正退出进程

当 `close_behavior=exit`：

- 点窗口关闭按钮：立即退出进程

当 `minimize_behavior=tray`：

- 点最小化按钮：隐藏到托盘

当 `minimize_behavior=taskbar`：

- 点最小化按钮：最小化到任务栏

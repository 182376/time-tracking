# 计时系统回滚清单

## 1. 触发条件

- [ ] 发布后出现系统进程再次进入统计结果
- [ ] 发布后出现同应用变体重复上榜
- [ ] 发布后出现统计总时长明显偏差

## 2. 代码回滚步骤

1. 回滚最近 tracking 相关提交（优先回滚标准化层变更）。
2. 重点确认文件：
  - `src/lib/processNormalization.ts`
  - `src/lib/services/sessionCompiler.ts`
  - `src/lib/db.ts`
  - `src/App.tsx`
3. 重新执行：
  - `npm test`
  - `npm run build`
  - `cargo test`

## 3. 数据回滚策略

当前版本使用“只读过滤 + 读时归并”，默认不改写历史数据：

- [ ] 无需 DB 数据回滚

若未来启用离线迁移脚本：

- [ ] 先恢复迁移前备份数据库
- [ ] 再回滚对应迁移版本代码

## 4. 回滚验收

- [ ] 应用排行、应用分布恢复可用
- [ ] 系统进程不在统计中
- [ ] 日志和测试结果无新增错误
